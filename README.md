# Expense Intelligence

An AI-powered expense tracking and financial insights platform with real-time analytics, budget management, and intelligent financial advice.

**Version**: 1.0.0 | **Status**: Production Ready

## 🎯 Features

- **User Authentication**: Secure JWT-based authentication with argon2 password hashing
- **Expense Tracking**: Record and categorize transactions with merchants and payment modes
- **Financial Insights**: Real-time analysis including spending trends, category breakdown, recurring subscriptions, anomaly detection, and financial health scoring
- **Budget Management**: Set monthly limits and receive alerts
- **AI Chat**: Ask questions about your finances and get personalized insights
- **Recurring Detection**: Automatically identify subscriptions
- **Monthly Summary**: AI-generated financial summaries

## 📋 Requirements

- Python 3.9+
- PostgreSQL 12+
- Node.js 16+ (frontend)

## 🚀 Quick Start

### Backend Setup (5 minutes)

```bash
cd backend
python -m venv .venv
source .venv/bin/activate  # Windows: .\.venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
# Edit .env with your PostgreSQL database URL
python -c "from app.database.db import init_db; init_db()"
python run.py
```

Backend runs at: `http://localhost:8000`

### Frontend Setup

```bash
cd frontend
npm install
npm run dev
```

Frontend runs at: `http://localhost:5173`

### Using Docker

```bash
docker-compose up -d
```

Starts PostgreSQL, FastAPI backend, and configures everything automatically.

## 🔌 API Endpoints

All endpoints require JWT authentication (except register/login).

### Authentication
- `POST /auth/register` - Register new user
- `POST /auth/login` - Login and get access token

### Transactions
- `POST /transactions/` - Add transaction
- `GET /transactions/` - Get user's transactions

### Insights
- `GET /insights/` - Get financial insights

### Budget
- `POST /budget/` - Set monthly budget limit

### AI Chat
- `POST /chat/` - Ask AI about your finances

### Example API Call
```bash
# Register
curl -X POST http://localhost:8000/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.com","password":"password123"}'

# Login
curl -X POST http://localhost:8000/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.com","password":"password123"}'

# Get insights (with token)
curl -X GET http://localhost:8000/insights/ \
  -H "Authorization: Bearer <your-token-here>"
```

**Interactive API docs**: `http://localhost:8000/docs` (auto-generated by FastAPI)

## ➕ Adding New Features

To add a new endpoint:

1. **Create service** in `app/services/feature_service.py`
2. **Create routes** in `app/api/feature_routes.py`  
3. **Update models** in `app/models/models.py` if needed
4. **Include router** in `app/main.py`:
   ```python
   from app.api import feature_routes
   app.include_router(feature_routes.router)
   ```

Example: New reporting endpoint with ~30 lines total, automatically tested and production-ready.

## 🐛 Troubleshooting

| Issue | Solution |
|-------|----------|
| Database connection error | Check `DATABASE_URL` in `.env`, ensure PostgreSQL running |
| CORS error | Update `CORS_ORIGIN` in `.env` with your domain |
| Auth token not working | Check `SECRET_KEY` is consistent across restarts |
| Port already in use | Change port: `python run.py --port 8001` |
